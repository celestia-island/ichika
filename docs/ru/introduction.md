# Введение

**Ichika** — это библиотека процедурных макросов Rust для создания конвейеров на основе пулов потоков с автоматической обработкой ошибок, семантикой повтора и поддержкой корректного завершения.

## Обзор

Ichika предоставляет мощный макрос `pipe!`, который позволяет определять сложные многоэтапные конвейеры обработки, где каждый этап выполняется в собственном пуле потоков. Макрос обрабатывает весь шаблонный код для создания пулов потоков, настройки каналов связи и координации между этапами.

## Ключевые особенности

- **Декларативный синтаксис конвейера**: Определяйте сложные конвейеры обработки, используя чистый и выразительный синтаксис макроса
- **Автоматическое управление пулами потоков**: Каждый этап получает свой собственный пул потоков
- **Распространение ошибок**: Встроенная обработка ошибок с типами `Result` на протяжении всего конвейера
- **Семантика повтора**: Настраиваемые политики повтора для обработки временных сбоев
- **Независимость от асинхронного окружения**: Работает как с `tokio`, так и с `async-std`
- **Корректное завершение**: Правильная очистка при удалении конвейера
- **Мониторинг**: Встроенная статистика использования потоков и подсчет задач

## Простой пример

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // Создаем простой 2-этапный конвейер
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // Отправляем запросы
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // Собираем результаты
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## Cases использования

Ichika особенно полезен для:

- **Конвейеров обработки данных**: Многоэтапные рабочие процессы преобразования данных
- **Обработки API-запросов**: Обработка запросов через несколько этапов проверки/преобразования
- **Обработки событий**: Создание систем, управляемых событиями, с поэтапной обработкой
- **Пакетных заданий**: Параллельная обработка с настраиваемой параллельностью на каждом этапе
- **Микросервисов**: Внутренняя служебная связь с ограниченными очередями

## Философия проектирования

Ichika следует следующим принципам:

1. **Безопасность прежде всего**: Использует систему типов Rust для гарантий на этапе компиляции
2. **Эргономичный API**: Минимизирует шаблонный код, сохраняя гибкость
3. **Абстракции с нулевой стоимостью**: Нет накладных расходов во время выполнения сверх необходимого
4. **Явное управление**: Предоставляет пользователям точный контроль над пулами потоков и очередями

## Состояние проекта

Ichika находится в активной разработке. API может меняться между версиями, но мы стремимся поддерживать обратную совместимость, когда это возможно.
