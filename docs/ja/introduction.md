# はじめに

**Ichika** は、スレッドプールベースのパイプラインを構築するためのRust手続き型マクロライブラリです。自動エラー処理、リトライセマンティクス、graceful shutdownサポートを備えています。

## 概要

Ichika は強力な `pipe!` マクロを提供し、各ステージが独自のスレッドプールで実行される複雑なマルチステージ処理パイプラインを定義できます。このマクロは、スレッドプールの作成、通信チャネルのセットアップ、ステージ間の調整に関連するすべての定型コードを処理します。

## 主な機能

- **宣言型パイプライン構文**: クリーンで表現力豊かなマクロ構文を使用して複雑な処理パイプラインを定義
- **自動スレッドプール管理**: 各ステージは専用のスレッドプールを取得
- **エラー伝播**: パイプライン全体に組み込まれた `Result` 型によるエラー処理
- **リトライセマンティクス**: 一時的な障害を処理するための設定可能なリトライポリシー
- **非同期ランタイムに依存しない**: `tokio` と `async-std` の両方をサポート
- **グレースフルシャットダウン**: パイプラインがドロップされるときの適切なクリーンアップ
- **モニタリング**: 組み込みのスレッド使用統計とタスクカウント

## シンプルな例

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // シンプルな2ステージパイプラインを作成
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // リクエストを送信
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // 結果を収集
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## 使用例

Ichika は以下の場合に特に役立ちます：

- **データ処理パイプライン**: マルチステージデータ変換ワークフロー
- **APIリクエスト処理**: 複数の検証/変換ステージを通じてリクエストを処理
- **イベント処理**: 段階的処理を備えたイベント駆動システムの構築
- **バッチジョブ**: ステージごとに構成可能な並行性を持つ並列処理
- **マイクロサービス**: 有界キューを持つ内部サービス通信

## 設計哲学

Ichika は以下の原則に従います：

1. **安全第一**: コンパイル時保証のために Rust の型システムを活用
2. **人間工学な API**: 柔軟性を維持しながら定型コードを最小化
3. **ゼロコスト抽象化**: 必要以上のランタイムオーバーヘッドなし
4. **明示的な制御**: スレッドプールとキューのきめ細かい制御をユーザーに提供

## プロジェクトの状態
