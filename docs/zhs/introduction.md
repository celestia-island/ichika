# 简介

**Ichika** 是一个 Rust 过程宏库，用于构建基于线程池的管道，具有自动错误处理、重试语义和优雅关闭支持。

## 概述

Ichika 提供了强大的 `pipe!` 宏，让您可以定义复杂的多阶段处理管道，其中每个阶段都在自己的线程池中运行。该宏处理了创建线程池、设置通信通道以及协调各阶段之间的所有样板代码。

## 主要特性

- **声明式管道语法**：使用清晰、富有表现力的宏语法定义复杂的处理管道
- **自动线程池管理**：每个阶段都有自己的专用线程池
- **错误传播**：通过管道内置的 `Result` 类型错误处理
- **重试语义**：可配置的重试策略，用于处理瞬态故障
- **异步运行时无关**：支持 `tokio` 和 `async-std`
- **优雅关闭**：当管道被丢弃时正确清理
- **监控**：内置的线程使用统计和任务计数

## 简单示例

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // 创建一个简单的 2 阶段管道
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // 发送一些请求
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // 收集结果
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## 使用场景

Ichika 特别适用于：

- **数据处理管道**：多阶段数据转换工作流
- **API 请求处理**：通过多个验证/转换阶段处理请求
- **事件处理**：构建具有分阶段处理的事件驱动系统
- **批处理作业**：每个阶段可配置并发度的并行处理
- **微服务**：具有有界队列的内部服务通信

## 设计理念

Ichika 遵循以下原则：

1. **安全第一**：利用 Rust 的类型系统提供编译时保证
2. **人体工学 API**：在保持灵活性的同时最小化样板代码
3. **零成本抽象**：除了必要的部分外没有运行时开销
4. **显式控制**：为用户提供对线程池和队列的细粒度控制
