# مقدمة

**Ichika** هي مكتبة ماكرو إجرائية من Rust لبناء خطوط أنابيب بناءً على مجموعات مؤشرات الترابط مع معالجة أخطاء تلقائية ودلالات إعادة المحاولة ودعم إيقاف التشغيل بأناقة.

## نظرة عامة

يوفر Ichika ماكرو `pipe!` قويًا يتيح لك تحديد خطوط أنابيب معالجة متعددة المراحل المعقدة حيث يتم تشغيل كل مرحلة في مجموعة مؤشرات الترابط الخاصة بها. يتعامل الماكرو مع كل رمز القوالب لإنشاء مجموعات مؤشرات الترابط وإعداد قنوات الاتصال والتنسيق بين المراحل.

## الميزات الرئيسية

- **صيغة خط الأنابيب التصريحية**: حدد خطوط أنابيب معالجة معقدة باستخدام صيغة ماكرو نظيفة ومعبرة
- **إدارة مجموعة مؤشرات الترابط التلقائية**: تحصل كل مرحلة على مجموعة مؤشرات الترابط المخصصة الخاصة بها
- **انتشار الخطأ**: معالجة أخطاء مدمجة مع أنواع `Result` في جميع أنحاء خط الأنابيب
- **دلالات إعادة المحاولة**: سياسات إعادة محاولة قابلة للتكوين للتعامل مع الأخطاء المؤقتة
- **مستقل عن وقت التشغيل غير المتزامن**: يعمل مع كل من `tokio` و `async-std`
- **إيقاف التشغيل بأناقة**: تنظيف مناسب عند إسقاط خط الأنابيب
- **المراقبة**: إحصائيات استخدام مؤشر الترابط المدمجة وعدد المهام

## مثال بسيط

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // إنشاء خط أنابيب بسيط من مرحلتين
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // إرسال الطلبات
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // جمع النتائج
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## حالات الاستخدام

Ichika مفيد بشكل خاص لـ:

- **خطوط أنابيب معالجة البيانات**: سير عمل تحويل البيانات متعدد المراحل
- **معالجة طلبات API**: معالجة الطلبات عبر مراحل متعددة من التحقق/التحويل
- **معالجة الأحداث**: بناء أنظمة مدفوعة بالأحداث مع معالجة مرحلية
- **المهام المجمعة**: المعالجة المتوازية مع التزامن القابل للتكوين لكل مرحلة
- **الخدمات المصغرة**: الاتصال الداخلي للخدمة مع قوائم محدودة

## فلسفة التصميم

يتبع Ichika هذه المبادئ:

1. **السلامة أولاً**: يستخدم نظام نوع Rust لضمانات وقت الترجمة
2. **API مريح**: يقلل من رمز القالب مع الحفاظ على المرونة
3. **تجريدات التكلفة الصفرية**: لا يوجد حمل وقت التشغيل بما يزيد عن الضروري
4. **تحكم صريح**: يمنح المستخدمين تحكمًا دقيقًا في مجموعات مؤشرات الترابط والقوائم

## حالة المشروع

Ichika قيد التطوير النشط حاليًا. قد تتغير API بين الإصدارات، ولكننا نسعى جاهدين للحفاظ على التوافق مع الإصدارات السابقة كلما كان ذلك ممكنًا.
