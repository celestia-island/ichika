# 簡介

**Ichika** 是一個 Rust 過程宏庫，用於構建基於線程池的管道，具有自動錯誤處理、重試語義和優雅關閉支援。

## 概述

Ichika 提供了強大的 `pipe!` 宏，讓您可以定義複雜的多階段處理管道，其中每個階段都在自己的線程池中運行。該宏處理了創建線程池、設置通信通道以及協調各階段之間的所有樣板代碼。

## 主要特性

- **聲明式管道語法**：使用清晰、富有表現力的宏語法定義複雜的處理管道
- **自動線程池管理**：每個階段都有自己的專用線程池
- **錯誤傳播**：通過管道內置的 `Result` 類型錯誤處理
- **重試語義**：可配置的重試策略，用於處理瞬態故障
- **異步運行時無關**：支援 `tokio` 和 `async-std`
- **優雅關閉**：當管道被丟棄時正確清理
- **監控**：內置的線程使用統計和任務計數

## 簡單示例

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // 創建一個簡單的 2 階段管道
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // 發送一些請求
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // 收集結果
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## 使用場景

Ichika 特別適用於：

- **數據處理管道**：多階段數據轉換工作流
- **API 請求處理**：通過多個驗證/轉換階段處理請求
- **事件處理**：構建具有分階段處理的事件驅動系統
- **批處理作業**：每個階段可配置併發度的並行處理
- **微服務**：具有有界隊列的內部服務通信

## 設計理念

Ichika 遵循以下原則：

1. **安全第一**：利用 Rust 的類型系統提供編譯時保證
2. **人體工學 API**：在保持靈活性的同時最小化樣板代碼
3. **零成本抽象**：除了必要的部分外沒有運行時開銷
4. **顯式控制**：為用戶提供對線程池和隊列的細粒度控制
