# 소개

**Ichika**는 자동 오류 처리, 재시도 의미론, 우아한 종료 지원을 갖춘 스레드 풀 기반 파이프라인을 구축하기 위한 Rust 절차형 매크로 라이브러리입니다.

## 개요

Ichika는 강력한 `pipe!` 매크로를 제공하여 각 단계가 자체 스레드 풀에서 실행되는 복잡한 다단계 처리 파이프라인을 정의할 수 있습니다. 이 매크로는 스레드 풀 생성, 통신 채널 설정, 단계 간 조정과 관련된 모든 상용구 코드를 처리합니다.

## 주요 기능

- **선언형 파이프라인 구문**: 깔끔하고 표현력이 풍부한 매크로 구문을 사용하여 복잡한 처리 파이프라인 정의
- **자동 스레드 풀 관리**: 각 단계는 전용 스레드 풀을 가짐
- **오류 전파**: 파이프라인 전체에 내장된 `Result` 타입을 통한 오류 처리
- **재시도 의미론**: 일시적인 장애 처리를 위한 구성 가능한 재시도 정책
- **비동기 런타임 무관**: `tokio` 및 `async-std` 지원
- **우아한 종료**: 파이프라인이 삭제될 때 적절한 정리
- **모니터링**: 내장된 스레드 사용 통계 및 작업 수

## 간단한 예제

```rust
use ichika::prelude::*;

fn main() -> anyhow::Result<()> {
    // 간단한 2단계 파이프라인 생성
    let pool = pipe![
        |req: String| -> usize {
            Ok(req.len())
        },
        |req: usize| -> String {
            Ok(req.to_string())
        }
    ]?;

    // 요청 전송
    pool.send("hello".to_string())?;
    pool.send("world".to_string())?;

    // 결과 수집
    while let Some(result) = pool.recv()? {
        println!("Got: {}", result);
    }

    Ok(())
}
```

## 사용 사례

Ichika는 다음과 같은 경우에 특히 유용합니다:

- **데이터 처리 파이프라인**: 다단계 데이터 변환 워크플로우
- **API 요청 처리**: 여러 검증/변환 단계를 통해 요청 처리
- **이벤트 처리**: 단계적 처리가 있는 이벤트 기반 시스템 구축
- **배치 작업**: 단계별로 구성 가능한 동시성을 갖는 병렬 처리
- **마이크로서비스**: 유계 큐가 있는 내부 서비스 통신

## 설계 철학

Ichika는 다음 원칙을 따릅니다:

1. **안전 우선**: 컴파일 시간 보장을 위해 Rust의 타입 시스템 활용
2. **인체공학적 API**: 유연성을 유지하면서 상용구 코드 최소화
3. **비용 없는 추상화**: 필요한 것 이상의 런타임 오버헤드 없음
4. **명시적 제어**: 사용자에게 스레드 풀과 큐에 대한 세분화된 제공 제공

## 프로젝트 상태

Ichika는 현재 활발하게 개발 중입니다. 버전 간에 API가 변경될 수 있지만 가능한 한 하위 호환성을 유지하도록 노력합니다.
